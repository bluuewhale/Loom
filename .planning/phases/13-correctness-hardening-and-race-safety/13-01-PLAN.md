# Phase 13 Plan 01: Correctness Hardening and Race Safety

## Objective

Prove through tests that `Update()` results satisfy all structural invariants and that concurrent use on distinct detector instances produces no data races.

## Context

- @graph/ego_splitting.go — Update() implementation with all invariants
- @graph/ego_splitting_test.go — Existing tests; add new tests here

## Tasks

### Task 1: assertResultInvariants + TestUpdateResultInvariants

type: auto

Add `assertResultInvariants(t, g, result)` helper that checks:
1. Every node in g appears in at least 1 community (NodeCommunities coverage)
2. NodeCommunities and Communities are mutually consistent (bidirectional)
3. No community index is out of bounds

Add `TestUpdateResultInvariants` table-driven test with 6 cases:
- empty delta
- single node addition (isolated)
- single edge addition (between existing nodes)
- multi-node batch addition (3 isolated nodes)
- node + edge addition together
- nil carry-forward fallback

### Task 2: TestEgoSplittingConcurrentUpdate

type: auto

Add `TestEgoSplittingConcurrentUpdate` that launches N goroutines each with their own `OnlineEgoSplittingDetector` instance calling `Update()` concurrently. Verifies `go test -race ./graph/...` passes with zero races.

## Verification

- `go test -race ./graph/...` passes
- All 6 invariant cases pass
- assertResultInvariants helper is reusable

## Success Criteria

1. assertResultInvariants helper added
2. TestUpdateResultInvariants with 6 cases passes
3. TestEgoSplittingConcurrentUpdate passes under -race
4. go test -race ./graph/... passes

**B. Cache-hit logic in reset() — replace the unconditional sort block (current lines 67-69):**

Current code:
```go
nodes := g.Nodes()
slices.Sort(nodes)
```

Replace with:
```go
nodeCount := g.NodeCount()
var nodes []NodeID
if len(st.sortedNodes) == nodeCount {
    // Warm-start with unchanged node set — reuse cached sorted slice.
    nodes = st.sortedNodes
} else {
    // Node set changed (or cold first use) — fetch, sort, and cache.
    nodes = g.Nodes()
    slices.Sort(nodes)
    st.sortedNodes = nodes
}
```

This works because in the ego-splitting Update() hot path, the global graph gains 1 node per update, but the INNER Louvain/Leiden calls operate on per-ego-net subgraphs whose node counts change. The real win is that the pooled state's cached sortedNodes from the prior call on the SAME subgraph (same ego node) will often match. For the typical 1-node-1-edge update, most ego-nets are unchanged.

**C. commStr delta patch in warm-start Step 4 (current lines 112-115):**

Current code (O(N) full rebuild):
```go
// Step 4: build commStr from CURRENT graph strengths (not from prior run).
for _, n := range nodes {
    st.commStr[st.partition[n]] += g.Strength(n)
}
```

Replace with a delta-patch approach. After Step 3 (remap), instead of iterating all N nodes:

1. Before clear(st.commStr) at the top of reset(), save the previous commStr if warm-start:
   - At the very beginning of reset(), before `clear(st.commStr)`, if `initialPartition != nil`, save `prevCommStr` by copying st.commStr entries.

2. After Step 3 (remap), remap prevCommStr keys through the remap table in O(|communities|):
```go
// Step 4: commStr delta patch — remap previous commStr through remap table,
// then patch only new/changed nodes instead of full O(N) rebuild.
//
// 4a: Remap previous community strengths to new compact IDs.
for oldC, str := range prevCommStr {
    if newC, ok := remap[oldC]; ok {
        st.commStr[newC] = str
    }
}
// 4b: Patch new nodes (not in initialPartition) — add their strength.
for _, n := range nodes {
    if _, ok := initialPartition[n]; !ok {
        st.commStr[st.partition[n]] += g.Strength(n)
    }
}
```

This is O(|communities|) + O(|new_nodes|) instead of O(N).

**Implementation detail for saving prevCommStr:**
Move the `clear(st.commStr)` call AFTER saving prevCommStr. The full reset() warm-start flow becomes:

```go
// At top of reset(), handle prevCommStr save BEFORE clearing:
var prevCommStr map[int]float64
if initialPartition != nil && len(st.commStr) > 0 {
    prevCommStr = make(map[int]float64, len(st.commStr))
    for k, v := range st.commStr {
        prevCommStr[k] = v
    }
}
clear(st.partition)
clear(st.commStr)  // now safe to clear
clear(st.neighborBuf)
// ... rest unchanged until Step 4
```

cold-start behavior, race safety, and all existing test invariants fully intact.

Current code at lines 68-69 (louvain) / 73-74 (leiden):

    var nodes []NodeID

        // Warm start, no new nodes: reuse cached sorted slice.
        nodes = st.sortedNodes

        nodes = g.Nodes()
        slices.Sort(nodes)
        st.sortedNodes = nodes
    }

g.NodeCount() should be used instead of len(g.Nodes()) to avoid allocating a second slice.
Check whether *Graph exposes NodeCount() — if not, use len(g.Nodes()) BUT assign to a
local variable first, THEN sort it for the cache. Do NOT call g.Nodes() twice.

──────────────────────────────────────────────
C. MODIFY reset() WARM-START — OPTIMIZATION B: commStr DELTA PATCH
──────────────────────────────────────────────

Current Step 4 (louvain line 113-115 / leiden line 118-120):
    // Step 4: build commStr from CURRENT graph strengths (not from prior run).
    for _, n := range nodes {
        st.commStr[st.partition[n]] += g.Strength(n)
    }

This is O(N) = 94K iterations on every warm Update call.

Replace with a two-sub-step approach that is O(|communities| + |new_nodes|):

Sub-step 4a: Remap previous community strengths through the compaction remap.
Sub-step 4b: Patch only the new-node singletons.

Implementation:

    // Step 4: build commStr via delta patch (warm-start only).
    // Sub-step 4a: remap prior commStr through the compaction remap.
    // prevCommStr captured BEFORE clear() at the top of reset() — see note below.
    for oldC, newC := range remap {
        if v, ok := prevCommStr[oldC]; ok {
            st.commStr[newC] = v
        }
    }
    // Sub-step 4b: patch new-node singletons (nodes absent from initialPartition).
    // Their old community had 0 strength (they didn't exist before), so we just add.
    for _, n := range nodes {
        if _, existed := initialPartition[n]; !existed {
            st.commStr[st.partition[n]] += g.Strength(n)
        }
    }

To make prevCommStr available:
- At the TOP of reset(), BEFORE `clear(st.commStr)`, capture a snapshot when in warm-start path:

    var prevCommStr map[int]float64
    if initialPartition != nil {
        prevCommStr = make(map[int]float64, len(st.commStr))
        for k, v := range st.commStr {
            prevCommStr[k] = v
        }
    }

Then clear(st.commStr) happens as before.

WHY THIS IS CORRECT:
- `remap` maps old community IDs (post-Step-1/2 assignment) → new contiguous IDs (Step 3).
- Prior commStr[oldC] = sum of g.Strength(n) for all nodes n where partition[n]=oldC.
- After compaction, these same nodes are now in newC, and their strengths haven't changed
  (graph edges didn't change for unaffected nodes). So commStr[newC] = prevCommStr[oldC].
- New nodes (not in initialPartition) got fresh singleton communities in Step 2.
  prevCommStr has no entry for those community IDs (they were generated as nextNewComm
  which is > all prior community IDs). After remap, their final compact IDs are correct,
  but their strength contribution is missing → patch in sub-step 4b.

EDGE CASE: First warm call when st.commStr is empty (pool just returned fresh state):
- prevCommStr will be an empty map (zero entries from the pool's zero-value commStr).
- Sub-step 4a copies nothing (empty remap from prevCommStr).
- Sub-step 4b patches only new-node singletons.
- But unaffected nodes also have no contribution! This means commStr is incomplete on the
  very first warm call from a fresh pooled state.

RESOLUTION: Only apply the delta patch when prevCommStr is non-empty (i.e. st.commStr had
data before the clear). If prevCommStr is empty, fall back to the O(N) full rebuild:

    if len(prevCommStr) > 0 {
        // Delta patch path (O(|communities| + |new_nodes|))
        for oldC, newC := range remap { ... }
        for _, n := range nodes {
            if _, existed := initialPartition[n]; !existed { ... }
        }
    } else {
        // First warm call on a freshly pooled state: full rebuild (O(N))
        for _, n := range nodes {
            st.commStr[st.partition[n]] += g.Strength(n)
        }
    }

This is still a net win: after the first warm call, all subsequent calls use the O(|communities|)
path. For BenchmarkEgoSplittingUpdate1Node1Edge the warmup call (before b.ResetTimer) ensures
the pool state is pre-populated, so every measured iteration uses the fast path.

# Phase 14 Plan 01: Reset Warm-Start commStr Delta Patch

## Objective

Add a `TestEgoSplittingUpdateAllocSavings` benchmark test that asserts incremental
`Update()` with a small delta allocates at least 2.0x fewer bytes than a full cold
`Detect()` on the same post-delta graph. Also verify `commStr` correctness in the
warm-start reset paths of `louvain_state.go` and `leiden_state.go`.

## Context

- `louvain_state.go` and `leiden_state.go` both have warm-start reset paths that
  rebuild `commStr` from the current graph's strengths.
- `ego_splitting.go` `Update()` uses incremental persona graph construction and
  warm-starts the global detector — allocations should be significantly less than
  a full `Detect()` call.
- The `BenchmarkUpdate_EmptyDelta` already asserts 0 allocs for empty delta.
- No test yet asserts alloc savings for non-empty delta `Update()` vs full `Detect()`.

## Tasks

### Task 1 (auto): Verify commStr warm-start correctness in reset()

Audit `louvain_state.go` reset() and `leiden_state.go` reset() warm-start paths
for correctness. Confirm that `commStr` is always rebuilt from the current graph's
`g.Strength(n)` values after the compaction remap (Step 4). No code changes
expected unless a bug is found.

**Done criteria:** `go test ./graph/ -race` passes.

### Task 2 (auto): Add TestEgoSplittingUpdateAllocSavings benchmark test

Add `TestEgoSplittingUpdateAllocSavings` to `graph/ego_splitting_test.go`.

The test:
- Builds Karate Club graph (34 nodes), runs `Detect()` to get prior state
- Adds node 34 + edge (0,34) to produce a minimal delta
- Measures allocs of `Update(g, delta, prior)` via `testing.Benchmark`
- Measures allocs of `Detect(g)` via `testing.Benchmark` on same post-delta graph
- Asserts `detect_allocs / update_allocs >= 2.0`

Also adds `BenchmarkEgoSplittingUpdate` companion benchmark for `go test -bench`.

**Done criteria:**
- `TestEgoSplittingUpdateAllocSavings` passes (≥2.0x alloc savings)
- `go test ./graph/ -race` passes

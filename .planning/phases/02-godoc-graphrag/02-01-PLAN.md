Scenario: Entity extraction from AI research papers -> graph construction -> Leiden clustering -> community context windows.

```go
// Simulate entity extraction results from an AI research paper corpus
reg := graph.NewRegistry()
g := graph.NewGraph(false)

// Entities: researchers, concepts, datasets
transformer := reg.Register("Transformer")
attention   := reg.Register("Attention")
bert        := reg.Register("BERT")
gpt         := reg.Register("GPT")
squad       := reg.Register("SQuAD")
glue        := reg.Register("GLUE")

// Co-occurrence / citation relationships
g.AddEdge(transformer, attention, 5.0)  // core architectural link
g.AddEdge(transformer, bert, 3.0)
g.AddEdge(transformer, gpt, 3.0)
g.AddEdge(bert, squad, 2.0)            // evaluation link
g.AddEdge(bert, glue, 2.0)
g.AddEdge(gpt, squad, 1.0)

// Detect research clusters
det := graph.NewLeiden(graph.LeidenOptions{Resolution: 1.0})
result, err := det.Detect(g)
if err != nil {
    log.Fatal(err)
}

// Group entities by community for RAG context windows
communities := make(map[int][]string)
for node, comm := range result.Partition {
    name, _ := reg.Name(node)
    communities[comm] = append(communities[comm], name)
}

// Each community becomes a context window for the LLM
for id, members := range communities {
    fmt.Printf("Community %d: %v\n", id, members)
}
```

Add a brief paragraph before the code explaining the pattern: "In a GraphRAG pipeline, entities extracted from documents become graph nodes and their relationships become edges. Community detection groups related entities into clusters, each serving as a focused context window for LLM queries."

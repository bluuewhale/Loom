**Create `runEgoNetsParallel` function** (place before `buildPersonaGraph`):

Signature: `func runEgoNetsParallel(g *Graph, nodes []NodeID, localDetector CommunityDetector) ([]egoNetResult, error)`

Implementation (collect-then-merge worker pool pattern from RESEARCH.md):
1. Allocate `results := make([]egoNetResult, len(nodes))` — index-disjoint slots, no mutex needed for results.
2. `nWorkers := runtime.GOMAXPROCS(0)` — matches core count.
3. Create buffered channel `work := make(chan int, len(nodes))`, fill with indices 0..len(nodes)-1, then `close(work)`.
4. Launch `nWorkers` goroutines, each: `for i := range work { v := nodes[i]; egoNet := buildEgoNet(g, v); if egoNet.NodeCount() == 0 { results[i] = egoNetResult{}; continue }; r, err := localDetector.Detect(egoNet); if err != nil { errMu.Lock(); if firstErr == nil { firstErr = err }; errMu.Unlock(); results[i] = egoNetResult{err: err}; continue }; results[i] = egoNetResult{partition: r.Partition} }`.
5. Use `var wg sync.WaitGroup`, `var firstErr error`, `var errMu sync.Mutex` for error propagation.
6. `wg.Wait()` then return `nil, firstErr` if firstErr != nil, else return `results, nil`.

CRITICAL: Workers must NOT write to shared maps (`personaOf`, `inverseMap`, `partitions`). Only write to `results[i]` (pre-allocated slice, index-disjoint = race-free).

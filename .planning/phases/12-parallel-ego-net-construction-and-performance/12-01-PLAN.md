## Objective

Introduce a goroutine worker pool for ego-net detection in both `buildPersonaGraph` (full Detect path) and `buildPersonaGraphIncremental` (Update path). Verify that:
1. `BenchmarkEgoSplitting10K` drops from ~1500ms/op to ≤300ms/op (ONLINE-10)

3. `BenchmarkUpdate1Edge` is ≥10x faster than `BenchmarkDetect` (ONLINE-09)

## Tasks

### Task 1: Parallel ego-net detection in buildPersonaGraph (goroutine pool)

**Type:** auto

**Goal:** Replace the serial node loop in `buildPersonaGraph` with a bounded goroutine worker pool. Each worker runs `localDetector.Detect(egoNet)` independently on its own copy of the local detector (CommunityDetector is not guaranteed thread-safe across calls with shared state). Collect results via channel; proceed to edge wiring only after all workers finish.

**Implementation notes:**
- Pool size: `runtime.GOMAXPROCS(0)` workers (auto-scales to available CPUs)
- Each worker receives a job `(nodeID, egoNet)` and sends back a result struct with `(nodeID, partition map[NodeID]int, err)`
- The local detector passed in `EgoSplittingOptions` may hold internal RNG state; each worker must use its own detector clone. Since `CommunityDetector` is an interface, clone by re-constructing via type switch (same pattern as `warmStartedDetector`). Add a helper `cloneDetector(d CommunityDetector) CommunityDetector`.
- Isolated nodes (egoNet.NodeCount()==0) are handled inline without spawning goroutines
- Edge wiring (Step 4-6) remains serial — it only reads the collected partitions map

**Done criteria:**
- `go test ./graph/... -run TestBuildPersonaGraph` passes (existing tests)
- `go test -race ./graph/...` passes

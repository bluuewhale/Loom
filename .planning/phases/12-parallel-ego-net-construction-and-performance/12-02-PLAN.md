**Key design decisions (from RESEARCH.md):**
- Edge endpoints for ONLINE-09: nodes 15 and 23 are chosen because they are peripheral with low degree (~2-3) in Karate Club. This minimizes `computeAffected` neighbor expansion, giving the incremental path maximum advantage. Per RESEARCH.md Pitfall 6, high-degree endpoints (e.g., node 0, degree 16) would expand `affected` to ~50% of the graph, likely failing the 10x target.
- Warmup call before `b.ResetTimer()` populates `sync.Pool` in underlying Louvain detectors.
- `prior` is computed once before `b.ResetTimer()` — only the Update/Detect under test runs in the loop (per Phase 5 benchmark decision).
- If ONLINE-09 fails 10x with nodes 15,23, try nodes with degree 1-2 (e.g., node 11 has degree 1). If still failing, document as calibration issue per RESEARCH.md Open Question 1.

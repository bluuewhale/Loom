**leiden.go** — Implement `func (d *leidenDetector) Detect(g *Graph) (CommunityResult, error)`:

1. Guard clauses — identical to `louvainDetector.Detect`:
   - `g.IsDirected()` -> return `ErrDirectedNotSupported`
   - `g.NodeCount() == 0` -> return empty `CommunityResult{}`
   - `g.NodeCount() == 1` -> return single-node partition `{node: 0}`, Q=0, Passes=1, Moves=0
   - `g.TotalWeight() == 0` -> return each node in own community, Q=0, Passes=1, Moves=0

2. Resolve zero-value options: `Resolution 0.0 -> 1.0`, `MaxIterations 0 -> unlimited` (use `d.opts.MaxIterations`), `Seed` passed to `newLeidenState`.

3. Initialize `origNodes`, `nodeMapping` (identity), `currentGraph = g`, `totalPasses = 0`, `totalMoves = 0`, `bestQ = -1.0`, `bestNodeMapping`, `bestSuperPartition`.

4. Main loop:
   a. Create `*louvainState` wrapper for `phase1` call:
      ```go
      ls := &louvainState{partition: state.partition, commStr: state.commStr, rng: state.rng}
      moves := phase1(currentGraph, ls, resolution, currentGraph.TotalWeight())
      state.partition = ls.partition
      state.commStr = ls.commStr
      ```
   b. **Leiden-specific: BFS refinement** (LEID-02):
      ```go
      state.refinedPartition = refinePartition(currentGraph, state.partition)
      ```
   c. Best-Q tracking using `refinedPartition` (not `partition`):
      ```go
      candidatePartition := reconstructPartition(origNodes, nodeMapping, state.refinedPartition)
      candidateQ := ComputeModularityWeighted(g, candidatePartition, resolution)
      ```
      Update `bestQ`, `bestNodeMapping`, `bestSuperPartition` if `candidateQ > bestQ`.
   d. Break conditions: `moves == 0`, or `maxIterations > 0 && totalPasses >= maxIterations`.
   e. Phase 3 aggregation using `refinedPartition` (LEID-03):
      ```go
      newGraph, superToRep := buildSupergraph(currentGraph, state.refinedPartition)
      ```
   f. nodeMapping update — CRITICAL: use `state.refinedPartition` (not `state.partition`):
      ```go
      commToNewSuper := make(map[int]NodeID, len(superToRep))
      for newSuper, rep := range superToRep {
          comm := state.refinedPartition[rep]
          commToNewSuper[comm] = newSuper
      }
      newMapping := make(map[NodeID]NodeID, len(nodeMapping))
      for orig, curSuper := range nodeMapping {
          comm := state.refinedPartition[curSuper]
          newMapping[orig] = commToNewSuper[comm]
      }
      nodeMapping = newMapping
      ```
   g. Update `currentGraph = newGraph`, reinitialize `state = newLeidenState(currentGraph, seed)`.
   h. Break if `currentGraph.NodeCount() <= 1`.

5. Final reconstruction: `reconstructPartition(origNodes, bestNodeMapping, bestSuperPartition)`, then `normalizePartition`, then `ComputeModularityWeighted`.

6. Return `CommunityResult{Partition, Modularity, Passes, Moves}`.

**refinePartition** function in leiden.go — BFS connectivity split:
- Group nodes by community into `commNodes map[int][]NodeID`
- Sort community IDs using insertion sort (no `sort` import) for determinism
- For each community, build `inComm map[NodeID]struct{}` set
- BFS from each unvisited node, only traversing edges where `e.To` is in `inComm` AND `e.To != cur` (skip self-loops)
- Each BFS connected component gets a unique `nextID` (incrementing int)
- Return `refined map[NodeID]int`

Use the authoritative BFS pattern from 03-RESEARCH.md Section "Full BFS Refinement".

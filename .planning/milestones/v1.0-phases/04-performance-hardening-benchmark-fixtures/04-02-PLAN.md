2. Create `graph/benchmark_test.go` (package graph):

   a. `generateBA(n, m int, seed int64) *Graph` — Barabasi-Albert preferential attachment graph generator:
      - Start with a complete graph on `m` nodes
      - For each new node from `m` to `n-1`, connect to `m` existing nodes with probability proportional to degree
      - Use seeded RNG for reproducibility
      - Result: ~n*m edges (10000*5 = ~50000 edges for 10K graph)

   b. Package-level var for reuse in benchmarks:
      ```go
      var bench10K *Graph
      func init() { bench10K = generateBA(10_000, 5, 42) }
      ```

   c. `BenchmarkLouvain10K(b *testing.B)`:
      ```go
      det := NewLouvain(LouvainOptions{Seed: 1})
      det.Detect(bench10K) // warmup — populate pool
      b.ResetTimer()
      b.ReportAllocs()
      for i := 0; i < b.N; i++ {
          det.Detect(bench10K)
      }
      ```

   d. `BenchmarkLeiden10K(b *testing.B)`: same pattern with NewLeiden.

   e. `BenchmarkLouvain10K_Allocs(b *testing.B)`: same as BenchmarkLouvain10K but explicitly checks that after warmup, allocs are minimized. (The -benchmem flag will report this.)

   f. `TestConcurrentDetect(t *testing.T)` (PERF-02):

      - Launch 4 goroutines, each calling `Detect` on its own graph 10 times
      - Use `sync.WaitGroup` to wait for all

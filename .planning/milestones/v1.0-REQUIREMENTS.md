## Milestone 1: Community Detection (v1 Requirements)

### Interface

- [x] **IFACE-01**: `CommunityDetector` 인터페이스 — `Detect(g *Graph) (CommunityResult, error)` 시그니처로 알고리즘 교체 가능
- [x] **IFACE-02**: `CommunityResult` 구조체 — `Partition map[NodeID]int`, `Modularity float64`, `Passes int`, `Moves int` 포함
- [x] **IFACE-03**: `LouvainOptions` 구조체 — `Resolution float64`, `Seed int64`, `MaxPasses int`, `Tolerance float64`
- [x] **IFACE-04**: `LeidenOptions` 구조체 — `Resolution float64`, `Seed int64`, `MaxIterations int`, `Tolerance float64`
- [x] **IFACE-05**: `NewLouvain(opts LouvainOptions) CommunityDetector` 생성자
- [x] **IFACE-06**: `NewLeiden(opts LeidenOptions) CommunityDetector` 생성자

### Louvain Algorithm

- [x] **LOUV-01**: Phase 1 local move — 각 노드를 이웃 커뮤니티로 이동 시 ΔQ 최대화
- [x] **LOUV-02**: Phase 2 supergraph compression — 커뮤니티를 수퍼노드로 축약, 자기 루프 보존
- [x] **LOUV-03**: 수렴 종료 — 이동이 없는 완전한 패스 후 종료 (tolerance 기반 ΔQ 비교)
- [x] **LOUV-04**: 올바른 modularity 수식 — 자기 루프 k_i_in 제외, resolution parameter 적용
- [x] **LOUV-05**: 엣지 케이스 — 빈 그래프, 단일 노드, 완전 단절 그래프, 2노드 그래프 모두 에러 없이 반환

### Leiden Algorithm

- [x] **LEID-01**: Phase 1 local move — Louvain Phase 1과 동일한 ΔQ 최적화
- [x] **LEID-02**: Refinement phase — 각 커뮤니티 내부 연결성 보장 (단절 커뮤니티 방지); 모든 커뮤니티를 매 iteration 정제
- [x] **LEID-03**: Phase 3 aggregation — 정제된 partition 기준으로 수퍼그래프 구성
- [x] **LEID-04**: Karate Club NMI ≥ 0.7 — ground-truth 2-커뮤니티 대비 Louvain보다 동등하거나 우수

### Performance

- [x] **PERF-01**: 10,000 노드 / ~50,000 엣지 그래프 기준 < 100ms (단일 고루틴, `-bench` 측정)
- [x] **PERF-02**: Concurrent-safe — 서로 다른 `*Graph` 인스턴스에 동시 `Detect` 호출 시 race condition 없음
- [x] **PERF-03**: `sync.Pool`로 `louvainState` 재사용 — 동일 크기 그래프 반복 처리 시 allocations 최소화
- [x] **PERF-04**: `go test -race` 통과 — 동시성 안전성 검증

### Testing & Benchmarks

- [x] **TEST-01**: Karate Club — Louvain, Leiden 모두 Q > 0.35 달성
- [x] **TEST-02**: Football network (115 노드, 613 엣지) 픽스처 추가 + NMI 검증
- [x] **TEST-03**: Polbooks 픽스처 추가 (105 노드, 441 엣지)
- [x] **TEST-04**: 8개 엣지 케이스 커버리지 — 빈 그래프, 단일 노드, 완전 단절, giant+singletons, 2노드, zero resolution, 완전 그래프, 자기 루프
- [x] **TEST-05**: `benchstat` 기반 성능 회귀 방지 테스트

## Milestone 2+ (v2 — Deferred)

### Graph Utilities

- **ALGO-02**: Label Propagation (초고속, 낮은 품질 트레이드오프)
- **ALGO-03**: Infomap

### GraphRAG Integration Helpers

- **GRAG-01**: 커뮤니티 계층 구조 (dendrogram) 반환
- **GRAG-02**: 커뮤니티별 요약 노드 선택 유틸리티

## Out of Scope

| Feature | Reason |
|---------|--------|
| 그래프 DB 커넥터 (Neo4j 등) | I/O 레이어는 라이브러리 외부; 단일 책임 원칙 |
| LLM 연동 / 임베딩 | 상위 레이어 책임; 알고리즘 라이브러리는 순수 연산 |
| 분산 처리 (멀티 머신) | 단일 프로세스 내 고루틴으로 충분 |
| 시각화 | 외부 도구 영역; 라이브러리 범위 외 |
| 영속성 / 직렬화 | 순수 인메모리; 직렬화는 호출자 책임 |
| Gonum 기반 구현 | Leiden 없음; 현재 Graph 구조체가 더 효율적 |
| GOEXPERIMENT=arena | golang/go#51317 — 무기한 보류 상태 |
